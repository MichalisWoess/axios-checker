5 Remediation Steps

  • 1
    Downgrade axios immediately
    Roll back to axios@1.14.0 or axios@0.30.3. Pin the version in package.json.
  • 2
    Remove the malicious dependency
    rm -rf node_modules/plain-crypto-js && npm install --ignore-scripts
  • 3
    Check for RAT artifacts on every machine that ran npm install
    See Section 2 above for OS-specific detection commands.
  • 4
    If RAT artifacts are found — assume full compromise
    Rotate all credentials: npm tokens, SSH keys, cloud API keys, database passwords, CI/CD secrets, crypto wallets.
  • 5
    Block C2 at the network level
    Block egress to sfrclak[.]com and 142.11.206.73 in your firewall/DNS.
  • 6
    Audit CI/CD pipelines
    Check if any build ran between Mar 31 00:21 UTC and the malicious versions being pulled. Those runners may be compromised.
  • 7
    Harden for the future
    Use npm ci --ignore-scripts in CI. Pin exact versions. Verify package provenance (SLSA attestations). Use lockfile integrity checks.
